‚óã Definition:
‚ñ† Non-relational databases, often termed NoSQL databases,
    diverge from the table-based structure. Instead, they primarily
    operate around objects or documents. There are no fixed table
    structures or rigid relationships between tables. One common
    format in NoSQL databases is the key-value pair model, similar
    to JSON (JavaScript Object Notation). An example of a
    non-relational database is MongoDB.
‚óã Object-Oriented Approach:
‚ñ† NoSQL databases work with collections of objects rather than
    structured tables.
‚ñ† They typically adopt a more flexible schema, allowing for easier
    scalability and adaptability.


Why Use NoSQL?

NoSQL databases are designed for:
    Scalability (handle massive amounts of data)
    Flexibility (no fixed schema)
    Speed (optimized for specific data access patterns)
    Handling unstructured or semi-structured data (like JSON, text, images, logs)
